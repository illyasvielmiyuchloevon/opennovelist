# PRD

## 1. 背景与目标

当前章节重写流程按固定 5 章切组，并在章节阶段继续依赖参考源章节正文。这会让“卷级资料适配”与“章节生产”之间缺少稳定的中间规划层，章节组数量也无法根据本卷实际推进节奏调整。

目标是在卷资料适配流程中前置生成整卷组纲计划和所有组纲，并完成组纲审核。章节重写阶段只读取已审核组纲计划，按动态章节组生成正文和审核，不再读取参考源章节正文。

## 2. 目标用户与使用场景

目标用户是使用本仓库进行中文网文改编、仿写和长篇连续生产的操作者。

核心场景：

- 从 `split_novel` 的参考源目录逐卷做资料适配。
- 卷资料审核通过后，在同一卷流程里生成整卷组纲计划和组纲。
- 后续按组纲计划顺序生成章节组正文、组审和卷级章节审核。

## 3. 核心需求

- 卷纲必须给出章节组推进指导，说明参考源章节范围的推进功能。
- 卷资料审核通过后继续生成整卷组纲计划、所有组纲和组纲审核。
- 组纲计划记录 `total_groups`、`total_chapters`、每组章节号、组纲路径和审核状态。
- 章节重写只按已审核组纲计划分组，不再回退固定 5 章。
- 章节组生成、组审、卷级章节审核不读取参考源章节正文。
- 已审核组纲在章节阶段只读冻结。

## 4. 用户流程

1. 运行 `novel_adaptation` 处理当前卷。
2. 生成卷资料并完成卷资料审核。
3. 自动进入整卷组纲计划生成、组纲文件生成和组纲审核。
4. 组纲审核通过后，当前卷写入 `processed_volumes`。
5. 运行 `novel_chapter_rewrite`。
6. 系统读取已审核组纲计划，按计划逐组生成正文、组审，最后执行卷级章节审核。

## 5. 功能范围

### 5.1 MVP 必须实现

- 新增组纲计划 manifest 与路径工具。
- 在 `novel_adaptation` 中接入组纲计划、组纲生成、组纲审核。
- 在 `novel_chapter_rewrite` 中以组纲计划作为唯一分组来源。
- 移除章节组生成、组审、卷级审核中的参考源正文依赖。
- 更新测试和使用文档。

### 5.2 可选增强

- 未来可增加旧工程迁移命令，辅助为历史项目补跑组纲计划。
- 未来可把遗留 `five_chapter` 函数名逐步迁移为中性命名。

### 5.3 明确不做

- 不为缺少新组纲计划的旧工程自动回退固定 5 章。
- 不在章节阶段重新生成或改写已审核组纲。
- 不改变 `split_novel` 的 50 章初始拆卷规则。

## 6. 输入与输出

输入：

- `split_novel` 参考源卷目录。
- 全局注入、卷级注入、卷纲和卷资料审核结果。

输出：

- `group_injection/<volume>_group_injection/00_group_outline_plan.md`
- `group_injection/<volume>_group_injection/<start>_<end>_group_injection/<start>_<end>_group_outline.md`
- `group_injection/<volume>_group_injection/<volume>_group_outline_review.md`
- 按动态组生成的正文、组审和卷级审核产物。

## 7. 非功能需求

- 性能：沿用现有缓存分层和 agent runtime，不引入额外全量扫描。
- 稳定性：缺少或未通过组纲计划时明确阻断。
- 安全：不新增密钥配置，不输出 API Key。
- 日志：进度输出说明当前使用组纲/卷级/全局注入，而不是参考源章节输入。
- 可维护性：组纲计划路径与 manifest 逻辑集中到 `novelist.core.group_outline_plan`。

## 8. 边界情况与异常处理

- 组纲计划为空时阻断。
- 组纲计划未审核通过时阻断。
- 组纲文件缺少标题或章节块时阻断。
- 组纲审核失败时不标记当前卷 processed，并记录阻塞问题。
- 章节阶段发现组纲问题时阻断，不在章节阶段修改组纲。

## 9. 验收标准

- 卷资料审核通过后不会立即标记 processed。
- 组纲生成能写出动态组数和每组组纲文件。
- 组纲审核通过后才标记 processed。
- 章节工作流从组纲计划读取动态分组。
- 没有组纲计划时章节工作流阻断，不回退固定 5 章。
- 章节组生成、组审、卷级审核 payload 不包含参考源正文。
- 目标测试和关键回归测试通过。

## 10. 假设与待确认事项

- 新书章节号按卷内连续编号，从 `0001` 到组纲计划的 `total_chapters`。
- 旧工程缺少组纲计划时需要补跑卷资料适配的组纲生成/审核阶段。
- 遗留函数名中的 `five_chapter` 暂保留为兼容 API，运行语义已改为动态组计划。
