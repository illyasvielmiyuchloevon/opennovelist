# AI 痕迹审查 Skill

## 概述

本 skill 对已完成的小说章节进行 AI 生成痕迹审查，**只出报告，不改原文**。审查报告按章保存，供写作模型读取后自行修改。

## 何时使用

- 每完成一章写作后
- 批量审查多个章节
- 写作模型修改后的复审

## 核心原则

- **只审查，不修改**——输出报告，不碰原文
- **保护小说叙事**：象征、隐喻、心理描写、判断句、破折号都是合法文学工具，不标记
- **宁可误报，不可漏报**——每一条标记都要附带判断理由

---

## 审查规则

### 第一优先级：核心 AI 模式

#### R1：「不是……而是……」句式

扫描所有否定+转折的对比句式：
- `不是 A，而是 B`
- `不是 A，是 B`
- `不仅是 A，更是 B`
- `不仅仅是 A，而是 B`
- `与其说是 A，不如说是 B`

**判定标准**（100% 标记，绝对禁止，无任何例外）：
所有否定+转折对比句式一律标记。不存在「可保留」的情况。即使删除后信息有缺失，也必须用非否定对比的方式重写来补回信息。

**报告中须附建议重写方向**：

> ⛔ **严禁使用任何否定对比句式作为替换方案。** 「不是A——是B」「不是A，是B」等变体仍然是 R1 违规。建议方向必须彻底消除否定对比框架。

| 方式 | 示例 |
|------|------|
| 转为单肯定 | `不是勇敢，而是绝望` → `那是绝望` |
| 转为双肯定 | `不是冷漠，而是克制` → `是克制，也是自我保护` |
| 转为具体描写 | `不是恐惧，而是敬畏` → `他的膝盖不自觉地弯了下去` |
| 直接保留 B | `不是风声，而是哭声` → `那是哭声` |
| ⛔ 禁止 | `不是勇敢，而是绝望` → ~~`不是勇敢——是绝望`~~（仍为否定对比） |

#### R2：稀疏排版 / 极短句

扫描目标：
- 独占一行且 ≤15 字的非对话句子
- 连续 2+ 个短句各占一行（诗歌体排版）
- 段落只有 1-2 句话且不是场景转换标记

**例外（不标记）**：
- 对话行（带引号的）
- 章节标题、场景分隔线
- 感叹/拟声词独行（如「砰。」「啪——」）
- 刻意的文学断句（需结合上下文判断）

#### R2B：AI 分镜断句

LLM 在生成叙事文本时，倾向于将连续动作拆解为独立的极短句，每句独占一行，模拟影视分镜脚本的「一帧一画面」效果。这不是英文翻译腔（R11 不覆盖），也不是选词问题（R10 不覆盖），而是 LLM 自身的生成节奏习惯。

**扫描目标**：

以「然后」「随后」「接着」「于是」开头（或仅含这类连接词），接极短动作/状态变化（≤12字），独占一行的句子。

**典型 AI 模式**：

| AI 写法（独行） | 问题 | 自然处理方式 |
|----------------|------|-------------|
| 然后暗下去。 | 状态变化孤立成句，像分镜指令 | 合并至前句尾部：「……都有了影子，随即暗了下来。」 |
| 然后她出现了。 | 「出现」无任何描写就独立成句，信息量为零 | 删除，直接进入出场描写 |
| 然后他松开了手。 | 动作与前后文割裂，制造不必要的戏剧停顿 | 合并至前段或后段：「他松开手，走到前面。」 |
| 接着，安静了下来。 | 环境变化独立成句 | 合并或改为具体描写 |

**判定标准**：

```
1. 该句是否以「然后/随后/接着/于是」开头？ → 否 → 按普通 R2 处理
2. 句子总长度 ≤12 字且独占一行？ → 否 → 不标记
3. 删除该句后，前后文是否仍然连贯？ → 是 → 标记（信息冗余）
4. 该句是否承载了无法从上下文推导的新信息？ → 否 → 标记
5. 同章内此模式出现 2 次以上？ → 提升为「系统性 AI 痕迹」
```

**单次出现的豁免**：如果全章仅出现 1 次，且该句确实在情感节奏上有不可替代的停顿功能（如高潮前的蓄力），可标记为「可保留」并附理由。

**与其他规则的关系**：
- 该句中的单字动词（如「暗下去」的「暗」）同时受 R9 审查
- 如果连续出现 3 次以上结构相同的分镜句，同时触发 R6（句式节奏）
- R2B 关注的是**叙事结构层面的 AI 习惯**，而非词汇或句法层面

### 第二优先级：AI 痕迹

#### R3：AI 高频词

衔接词类（通常应删除）：
- 「此外」「与此同时」「值得注意的是」「综上所述」
- 「然而」连续出现 2 次以上

修饰词类（应替换为具体描述）：
- 「至关重要」「深入探讨」「多元化」「错综复杂」「不可或缺」

#### R4：同义词刻意轮换

同一段内对同一概念使用 3+ 个不同近义词。小说中合理的重复是节奏感，刻意的近义词轮换才是 AI 味。

#### R5：填充短语

- 「在某种程度上」「从某种意义上说」「在这个过程中」
- 「为了实现这一目标」「在当今社会」「在一定程度上」

### 第三优先级：节奏与结构

#### R6：句式节奏问题

- 连续 3 个句子结构相同（如全是主谓宾）
- 连续 3 个句子长度接近（±5 字以内）
- 段尾连续使用相同句式收束

#### R7：三段式 / 排比过度

- `A、B 和 C` 连续出现多次
- 三个并列短句一组一组出现
- 偶尔的排比三是正常的，连续多次出现才标记

### 第四优先级：Tokenizer 残影

#### R9：非自然单音节截断

现代汉语以双音节词为主。AI 的分词器（tokenizer）将中文切分为 subword token，在生成时有时只输出一个字，而自然中文会用两个字的完整词。这导致文字读起来像被「截断」了——不是文言文的精炼，而是一种机械的生硬感。

**核心判定**：在当前语境中，母语中文读者是否会自然使用双音节词？如果是，则单音节用法为 tokenizer 残影。

> ⚠️ **本规则与其他规则不同**：R9 采用「宁可多报，不可漏报」原则。Tokenizer 截断具有隐蔽性，读者难以说清哪里怪但就是觉得不自然，因此审查时应积极标记所有可疑单音节用法，由写作模型判断是否采纳。

**高频截断词表**（仅为常见示例，非穷举，审查时应依语感判断）：

| 类型 | AI 截断形式 | 自然形式（示例） | 说明 |
|------|-----------|----------------|------|
| 动作动词 | 他**压**下情绪 | 他**压制**住情绪 / 他**压抑**着情绪 | 「压」独立做谓语不自然 |
| 动作动词 | 她**避**开目光 | 她**躲避**开目光 / 她**避开**了目光 | 缺少补语时不自然 |
| 动作动词 | 他**阻**了一下 | 他**阻止**了一下 / 他**阻挡**了一下 | 「阻」单用是文言 |
| 动作动词 | 他**挡**在前面 | 他**挡**在前面 ✅ | 这个OK——「挡」可独立使用 |
| 感官动词 | 他**凝**着远方 | 他**凝视**着远方 | 感官动词截断 |
| 感官动词 | 她**嗅**到气味 | 她**闻**到气味 / 她**嗅**到了气味 | 缺「到/了」时不自然 |
| 心理动词 | 他**惧**了 | 他**恐惧**了 / 他**害怕**了 | 「惧」单用是文言 |
| 心理动词 | 她**悔**了 | 她**后悔**了 / 她**悔恨**了 | 「悔」单用不自然 |
| 心理动词 | 他**疑**着 | 他**怀疑**着 / 他**疑惑**着 | 「疑」单用太文言 |
| 情绪形容词 | 一阵**怒**意 | 一阵**愤怒** / 一股**怒意** ✅ | 「怒意」是合法词组 |
| 情绪形容词 | 他感到**惊** | 他感到**惊讶** / 他**吃了一惊** | 「惊」不能做「感到」的宾语 |
| 状态形容词 | 四周很**寂** | 四周很**寂静** / 四周很**安静** | 「寂」单用不成立 |
| 状态形容词 | 他很**疲** | 他很**疲惫** / 他很**疲倦** | 「疲」不能做「很」的补语 |

**自然单音节词表（不标记）**：

以下单字动词/形容词在现代汉语中自然使用，不应被误报：
- 基础动作：看、说、想、走、跑、站、坐、笑、哭、叹、吃、喝、打、踢、抓、拉、推、扔
- 基础状态：好、坏、大、小、多、少、快、慢、高、低、冷、热、亮、暗
- 常见组合中的单字：挡在、靠着、躺下、蹲着、趴在、贴着

**判定流程**：

```
1. 该单字词是否在「自然单音节词表」中？ → 是 → 不标记
2. 该单字词是否后接补语/助词形成完整结构？
   （如「压下」「挡住」「避开」→ 结构完整）→ 是 → 不标记
3. 该单字词如果替换为双音节词，读感是否更自然？ → 是 → 标记
4. 该语境是否为刻意的文言/古风/诗化表达？ → 是 → 不标记
5. 不确定 → 标记为「疑似」（宁可多报，不可漏报）
```

**报告中须附**：原文、自然双音节替换建议、判定理由。

### 第五优先级：词汇与句法层洗涤

#### R10：英文选词污染与机械补音节

现代汉语以双音节词为主。R9 解决的是 tokenizer 把双音节词截断成单字的问题；R10 解决的是**修复截断后选词仍然不自然**的问题——根源是英文训练语料对中文动词搭配的污染。

**R10A. 机械补音节**：

单字被指出后只是硬凑一个双音节词，但选词仍然不是母语者会用的动词搭配。

**R10B. 英文动词直映射**：

英文某些动词在技术/日常语境中自然，但直译成中文形成不符合母语习惯的搭配。

**高频错误词表**（常见示例，非穷举）：

| 错误（不自然的表达） | 自然的中文 | 英文原型 |
|---|---|---|
| 抓取证据、抓证据 | 寻找证据、收集证据、搜集依据 | grab evidence |
| 拉取数据 | 获取数据、读取数据 | pull data |
| 喂入参数、喂参数 | 传入参数 | feed parameters |
| 吐出结果、吐结果 | 输出结果、返回结果 | spit out results |
| 灌入数据、灌数据 | 写入数据、导入数据 | dump / pour data |
| 捞出来、捞回来 | 取出、提取、恢复 | fish out / retrieve |
| 补一刀 | 补充一点、提供更多细节 | 非直译，但属于口语替代 |
| 怼接口 | 对接接口、联调接口 | 不固定 |
| 盘一下问题 | 分析问题、梳理问题 | 不固定 |
| 拆一下这个对象 | 分析这个对象、拆分这个对象、解析这个对象 | break down |

**R10C. 语境失配选词**：

AI 选择了一个语法上合法、甚至在 R9 豁免表内的词，但在**当前具体语境**中母语者会选完全不同的词。根源不是英文污染（R10B），也不是截断修复（R10A），而是 AI 的概念激活路径与母语者不同——AI 从语义空间里选了一个「技术上最近」的词，但母语者是从「这个场景通常怎么说」来选词的。

**与 R9 的关系**：R9 豁免表中的词（如「暗」「亮」「冷」「热」）在静态/形容词用法中确实自然（「很暗」「太冷了」）。但当这些词被用作**动态变化动词**描述具体事件时，可能不再是母语者的自然选择。R10C 专门捕捉这类「R9 放行但实际不自然」的情况。

**高频语境失配词表**（常见示例，非穷举）：

| 语境 | AI 选词 | 母语者自然选词 | 说明 |
|------|--------|--------------|------|
| 光线消散/熄灭 | 暗下去、暗了 | 灭了、黑了下来、光散了 | 「暗」做静态形容可以，做动态消散动词不自然 |
| 声音消失 | 静下去、静了 | 安静了、没声了、声音断了 | 「静」做状态可以，做瞬时变化动词不自然 |
| 温度骤降感受 | 冷下去 | 凉了、冷了下来、一阵寒意 | 「冷下去」偏书面/翻译腔 |
| 情绪突然转变 | 沉下去（表情沉下去） | 沉了下来、脸一沉、脸色变了 | 「沉下去」带有物理下坠感，用于表情不自然 |
| 力量/能量耗尽 | 弱下去 | 没力气了、虚了、软了 | 「弱」做动态变化不自然 |

**R10D. 虚词缺失与万能后缀**：

AI 在生成中文时经常丢失让句子自然流畅的虚词（副词、助词、语气词），或用万能抽象后缀代替具体名词。这两类问题的根源相同：AI 优先生成「语义核心」，而忽略中文母语者凭语感自动补上的功能性词语。

**R10D-1. 虚词/语气词缺失**：

AI 生成的句子语法正确，但缺少母语者会自然带上的虚词，导致句子读起来生硬、机械。

| AI 写法 | 自然写法 | 缺失的虚词 | 说明 |
|--------|---------|-----------|------|
| 但不够 | 但**还**不够 | 还（程度副词） | 「还」暗示「虽然有进步但仍未达标」的递进语气 |
| 他站在那 | 他站在那**儿** | 儿（词缀） | 口语中「那儿」比「那」自然 |
| 没时间想 | 没时间**去**想 / 没**来得及**想 | 去/来得及 | 缺少动作指向性 |
| 这是问题 | 这**倒是**个问题 / 这**确实**是个问题 | 倒是/确实 | 缺少语气层次 |
| 声音低 | 声音**很**低 / 声音低**了下来** | 很/了下来 | 裸形容词做谓语在中文里不自然 |

**R10D-2. 万能后缀/分类词**：

AI 偏好用抽象的分类后缀（型、式、级、类、感）代替具体名词，产生一种「设定集」或「游戏面板」的既视感。

| AI 写法 | 自然写法 | 说明 |
|--------|---------|------|
| 骑士基础**型** | 骑士基础**剑术** | 「型」是万能分类词，不如具体说「剑术」「步法」「架势」 |
| 攻击**型**圣光 | 攻击**性**圣光 / 用于攻击的圣光 | 「型」做后缀偏产品分类 |
| 搜索**型** | 搜索**用的** / 用来搜索的 | 同上 |
| 高强度**级**排查 | 高强度排查 | 「级」多余，不增加信息 |

**判定流程**：

```
1. 该动词/形容词搭配是否为中文母语者在该语境中自然会用的表达？ → 是 → 不标记
2. 该表达是否可以追溯到英文动词的直接映射？ → 是 → 标记为 R10B
3. 该双音节词虽然消除了单字问题，但搭配仍然不自然？ → 是 → 标记为 R10A
4. 该词在 R9 豁免表中但用法从静态转为动态/事件描述，母语者会选不同的词？ → 是 → 标记为 R10C
5. 句子缺少母语者会自然补上的虚词/语气词，或使用了万能抽象后缀代替具体名词？ → 是 → 标记为 R10D
6. 该表达在特定行业/亚文化中已经固化为术语？ → 是 → 不标记
7. 不确定 → 标记为「疑似」
```

**报告中须附**：原文、自然中文替换建议、子类编号（R10A-R10D）、判定理由。

#### R11：英文句法逻辑写中文

中文输出不得读起来像英文逐词翻译。以下模式全部扫描。

**R11A. "让我们"开头**：

- 扫描：`让我们先看看……`、`让我们来解释……`、`让我们一起……`
- 判定："Let's" 是英文口语习惯，直译"让我们"在中文里做作
- 建议方向：直接用祈使句或陈述句。如`让我们先看这个函数` → `先看这个函数`

**R11B. "如果……的话"滥用**：

- 扫描：每个假设句都用完整的`如果……的话`框架
- 判定：中文假设句不需要每次用完整框架，尤其连续出现时
- 建议方向：保留一种即可。如`如果你想要更好的性能的话` → `想要更好的性能，可以……`

**R11C. 生硬否定对仗**：

- 扫描：`它不是不能X，而是Y` 这类 `It's not that X, but rather that Y` 的直译
- 判定：中文里这类句式应改为更自然的让步句式
- 注意：与 R1 有交集。R1 扫描所有 `不是A而是B`；R11C 专门针对英文逻辑硬套的嵌套否定转折
- 建议方向：改成中文让步句。如`它不是不能解开，而是没有东西可以解开` → `问题不在于解不开，而在于根本没有可解的东西`

**R11D. 被动语态滥用**：

- 扫描：`被认为是……`、`被广泛地使用`、`被发现……`、`被证明……`
- 判定：中文很少使用被动句，英文被动直译使句子别扭
- 例外：`被打`、`被骂` 等确实需要被动标记的事件不标记
- 建议方向：改为主动句或无标记句。如`这个方法被广泛地使用` → `这个方法应用很广`

**R11E. "的"字叠罗汉**：

- 扫描：连续 3 个或以上`的`字修饰构成的定语链
- 判定：英文前置定语链不能直接映射成中文连续堆叠的"的"字结构
- 建议方向：超过两个"的"就该拆句。如`这是一个被广泛使用的用于处理数据的高效的工具` → `这个工具处理数据很高效，应用也很广`

**R11F. 无主语空降**：

- 扫描：`值得注意的是`、`需要指出的是`、`可以观察到`、`重要的是`
- 判定：这类前缀没有信息增量，只拖慢表达
- 注意：与 R3 / R5 有交集。R3/R5 把这类词标为 AI 高频词或填充短语；R11F 从句法层面解释其根源是英文 `It is worth noting that...` 结构
- 建议方向：直接删除前缀，陈述事实本身

**R11G. 名词化倾向**：

- 扫描：`对该问题的解决方案的实现`、`对数据的处理的结果`
- 判定：英文习惯把动词名词化（nominalization），中文应保持动词活性
- 建议方向：还原为动宾结构。如`对该问题的解决方案的实现` → `实现这个方案` / `解决这个问题`

**判定流程**：

```
1. 该句式是否为角色对话中刻意的口语/翻译腔模拟？ → 是 → 不标记
2. 该句式是否在当前语境中读起来像英文直译？ → 是 → 标记对应子类
3. 同一类型在全章出现 3 次以上？ → 提升为「系统性问题」
4. 不确定 → 标记为「疑似」
```

**报告中须附**：原文、子类编号（R11A-R11G）、自然中文重写建议、判定理由。

### 可选规则：R8 无用细节清除

> **默认关闭。** 启动审查时须询问用户是否开启。

开启时须告知：
> ⚠️ 此功能会标记大量细节描写为「建议删除」，有可能误伤你认为重要的内容。需要读取章节的副文本（大纲/人设/主旨）才能准确判断。

#### R8 开启时的前置要求

获取以下上下文（至少其一）：
- 章节所属的大纲/场景规划（本章主旨）
- 人物设定/人设文件
- 用户口头说明的本章重点
- 无法获取时：基于正文自行推断主旨，报告中标注「主旨为推断」

#### R8 四类保护对象（绝不标记）

1. **战斗/动作场景中的描写**——招式、走位、伤势、力量对比
2. **性爱/亲密场景中的描写**——身体反应、感官、情绪
3. **关键台词附带的动作**——角色说出重要台词时的直接伴随动作
4. **人物魅力/外观的直接描写**——容貌、体态、气质、着装

#### R8 高危扫描目标

**R8A. 无功能微动作**：
- 指尖泛白/发紧/收紧、手腕上翻/手背轻抬
- 眉头微蹙（非关键转折处）、嘴角微扬/微抿（非关键转折处）
- 睫毛颤动/眼睫低垂、喉结滚动/下颌绷紧

**R8B. 游离环境/物品碎片**：
- 与剧情无关的风吹描写（衣角、发丝、纸页）
- 装饰性光影变化（阳光穿过窗帘、月光洒地面——除非氛围是本段核心）
- 不推动剧情的天气描写

**R8C. 重复情绪外化**：
- 同一情绪在一段内用 2+ 种身体反应表达 → 标记保留最强的那个，其余为冗余

#### R8 判定流程

```
1. 属于四类保护对象？ → 是 → 不标记
2. 删掉后本段核心信息/情绪/节奏是否受损？ → 是 → 不标记
3. 是否建立后文伏笔或因果链？ → 是 → 不标记
4. 以上都不是 → 标记为「建议删除」
```

### 第六优先级：世界观一致性

#### R12：世界观术语一致性

AI 缺乏对作品已确立术语体系的记忆，容易出现两类问题：截短已确立的术语，或自造不在世界观内的新术语。这不是语言层面的问题（R9/R10 不覆盖），而是**叙事一致性**的问题。

**R12A. 术语截短**：

AI 将已在前文确立的完整术语截短为更短的形式，导致指代模糊或前后不一致。

| AI 写法 | 应为 | 说明 |
|--------|------|------|
| 画**阵** | 画**法阵** / 画**召唤阵** | 同章其他位置用了「召唤阵」，此处截短为「阵」，丢失了类型信息 |
| 用了**光** | 用了**圣光** | 世界观术语是「圣光」，不能随意缩略 |
| 释放**术** | 释放**法术** / 释放**咒语** | 「术」单用过于模糊 |

**R12B. 术语自造**：

AI 自行组合出听起来像游戏/设定集的复合术语，但这些术语不在作品已确立的世界观词汇中。

| AI 自造 | 应使用的世界观术语 | 说明 |
|--------|-----------------|------|
| 绝对压制场 | 圣光范围 / 圣光覆盖区域 | 「绝对压制场」像游戏技能名，世界观里没有这个概念 |
| 魔力反冲波 | 魔力反噬 | 世界观已确立的术语是「反噬」 |
| 灵魂共振频率 | ——（不存在） | AI 凭空发明的伪设定 |

**判定流程**：

```
1. 该术语是否在前文/世界观文件中已确立完整形式？
   → 是，且当前使用了截短形式 → 标记为 R12A
2. 该术语是否在世界观文件中存在？
   → 否，AI 自造的 → 标记为 R12B
3. 该截短/新造是否为角色对话中的口语简称？
   → 是（如角色习惯性简称）→ 不标记
4. 不确定 → 标记为「疑似」，附注需核查世界观文件
```

**前置要求**：审查 R12 时需要读取世界观文件（如 `02-worldbuilding.md`、`03-cast-bible.md`）以确认已确立的术语体系。如无法获取，基于正文内部的术语一致性进行判断，报告中标注「世界观文件未参考」。

**报告中须附**：原文、正确术语、出处（同章/世界观文件）、判定理由。

## 写作模型的使用方式

写作模型收到 AI 痕迹审查报告后：

1. 读取 `reviews/第X章-ai-audit.md`
2. 按「必须处理」逐条修改
3. 视情况采纳「建议处理」
4. 修改时遵循原始 Humanizer 的处理策略（R1 重写方式、R2 合并方式等）
5. 修改完成后在 `logs/writing-log.md` 记录

## 与 external-review-process 的关系

本 skill 与 `external-review-process.md` 定义的外部审稿是**独立并行**的两个审查流程：

- **外部审稿**（external-review）：关注文章质量、文风偏移、叙事评价——由审稿模型执行
- **AI 痕迹审查**（anti-ai-audit）：关注 AI 生成模式的识别和清除——由本 skill 执行

两者可以同时进行，报告分别存储：
```
reviews/
├── 第一章-review.md       ← 外部审稿报告
├── 第一章-ai-audit.md     ← AI 痕迹审查报告
└── ...
```

写作模型应先读取两份报告，综合修改。如果两份报告对同一处文字有冲突建议，以**外部审稿报告**为准（因为外部审稿关注的是文学质量，AI 痕迹清除不应以牺牲文学性为代价）。
