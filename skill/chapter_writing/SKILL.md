# 写作规范 Skill

## 概述

本 skill 用于当前项目的“完整章节仿写”阶段，目标是让模型在保留参考源章节功能映射的前提下，写出符合新书设定、文风稳定、能通过后续审核的正文。

本 skill 是正文生成阶段的主写作规则，不负责审核，也不负责状态文档更新。

## 核心原则

- 只写当前章正文，不输出解释、提纲、分析、总结或额外说明。
- 参考源只用于功能映射与节奏参照，不得照抄原书专有名词与可识别表达。
- 正文必须服务于新书当前阶段的世界观、人物、剧情与文风，而不是把参考源换名后重新抄一遍。
- 文风必须稳定，不得随机发挥成另一种腔调。

## 仿写硬规则

### 一、禁止直接沿用参考源可识别信息

在正文中，不得原样使用参考源的下列内容：

- 章节标题
- 人物名
- 地点名
- 势力名 / 宗门名 / 组织名
- 事件名
- 物品名 / 功法名 / 技能名 / 招式名 / 术语名
- 境界名 / 等级名 / 数值体系 / 具体数值 / 奖励数值 / 百分比 / 排名数字

如果参考源使用了某个强识别性的设定元素，必须改写成新书体系下的等效信息，而不是做同义替换式伪改名。

### 二、功能映射，不做表面换皮

正文仿写时要保留的是“这一章在整本书里的作用”，例如：

- 推进哪条剧情线
- 暴露哪个人物状态变化
- 引入哪个冲突
- 回收或推进哪个伏笔
- 制造哪个爽点
- 用什么方式收尾和挂钩

不要把仿写理解成：

- 参考源场景逐句改写
- 参考源桥段逐段平移
- 原人设 / 原世界观 / 原爽点结构的机械复刻

### 三、优先遵循新书上下文

正文必须同时服从以下信息：

- 当前章章纲
- 当前卷卷级大纲
- 全书大纲
- 世界观设计
- 文笔写作风格文档
- 当前人物状态卡 / 人物关系链 / 剧情进程 / 世界模型 / 世界状态

如果参考源和新书上下文存在冲突，以新书上下文为准。

## 写作执行规范

### 篇幅与节奏

- 正文字数应贴近当前参考章，不得无故扩写。
- 场景数量、冲突层级、节奏快慢、推进密度应尽量贴近参考章。
- 如果参考章是短促推进型，就保持短促。
- 如果参考章是对话驱动型，就保持相近的对话密度。
- 不要额外补写解释性段落、世界观说明、重复心理复述、总结性抒情来硬性扩字。

### 文风与结构

正文必须贴合文笔写作风格文档中的这些维度：

- 爽点铺垫
- 剧情转折
- 叙事节奏
- 情节结构
- 符号的使用习惯
- 段落分割
- 对话密度
- 句长
- 收尾方式

如果参考章本身在这些维度上表现很明确，应优先贴近参考章的实际写法，而不是抽象地“模仿一种风格”。

### 语言约束

- 不要写成总结式、解释式、AI 常见腔调的正文。
- 不要堆砌华丽意象、对称排比、诗化抒情。
- 不要出现审核 skill 会重点打回的典型 AI 痕迹。
- 用词要服务于小说叙事，不要写成设定说明书、游戏面板说明或数据库条目。

## 正文生成时的工作顺序

1. 先确认当前章章纲要完成什么剧情功能。
2. 再对照参考章，确认本章大致篇幅、场景数、冲突节奏、收尾功能。
3. 再对照全局与卷级注入，确认本章不能违背的人设、世界观与剧情进程。
4. 在新书体系下重建本章正文。
5. 最后检查是否仍残留参考源原名词、原标题、原数值或原事件标签。

## 自检清单

提交正文前，至少自检以下事项：

- 是否还残留参考源的人名、地名、势力名、事件名、物品名、数值等
- 是否只是对参考章做了表面换皮
- 是否无端扩写，导致篇幅和节奏明显偏离参考章
- 是否偏离文笔写作风格文档
- 是否与当前人物状态、剧情进程、世界模型和世界状态冲突
- 是否收尾方式正确，能服务下一章推进

## 使用边界

本 skill 只用于“完整章节仿写正文”。

不要用于：

- 章纲生成
- 状态文档更新
- 章级审核
- 组审查
- 卷级审核
